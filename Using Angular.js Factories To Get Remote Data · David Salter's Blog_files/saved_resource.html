                                            Jeremy Likness • 2 years ago

                                            I agree. If you are just binding the result to the $scope object, angular is smart enough to understand a promise and defer the promise then bind the data when it appears. So, instead of doing a callback, you can literally just return the promise itself and set it to the $scope, like this: $scope.fruits = fruitsFactory.getFruitsAsync() and then in the factory return $http.get('fruits.json');
